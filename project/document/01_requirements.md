## 2. 目的

Markdown ファイル（`.md`）を**表示・編集・保存**できるスタンドアロンデスクトップツールを作成する。

- 対象ユーザーは Windows ユーザーであり、Python 等の開発環境の知識がなくても使用できること。
- インストール作業不要で、`.exe` ファイルをダブルクリックするだけで即時利用できること。

---

## 3. スコープ

### 3.1 対象

| # | 対象範囲 |
|---|----------|
| 1 | Markdown ファイル（`.md`）の閲覧 |
| 2 | Markdown ファイルの編集（CodeMirror エディタ） |
| 3 | Markdown ファイルの保存（読み込み時エンコードで書き込み） |
| 4 | ユーザー指定フォルダ配下のファイルツリー表示 |
| 5 | 新規ファイル・フォルダ作成 |
| 6 | ファイル・フォルダの削除 |
| 7 | ファイル・フォルダ名の変更 |
| 8 | Markdown リアルタイムプレビュー（marked.js） |
| 9 | Windows 10 / 11 向け `.exe` 単一ファイル配布 |
| 10 | ドラッグ＆ドロップ操作を禁止し、ドロップされたファイルを読み込まないこと |

### 3.2 非対象

| # | 非対象範囲 | 理由 |
|---|------------|------|
| 1 | `.md` 以外のファイル形式の編集 | 議事録にて `.md` のみと明記 |
| 2 | 外部サーバー・クラウドとの連携 | 完全ローカル動作が前提 |
| 3 | macOS / Linux 対応 | Windows 10/11 のみ対象 |
| 4 | 複数ファイルの同時編集（マルチタブ） | 議事録に記載なし、初版スコープ外 |
| 5 | バージョン管理（Git 統合） | 議事録に記載なし、初版スコープ外 |

## 5. 機能要件

### 5.1 入力要件

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-IN-01 | フォルダ指定 | ユーザーが任意のローカルフォルダパスを指定できること |
| 2 | REQ-IN-02 | ファイル選択 | ファイルツリーから `.md` ファイルをクリックして選択できること |
| 3 | REQ-IN-03 | テキスト入力 | エディタ（CodeMirror）上で Markdown テキストを直接入力・編集できること |
| 4 | REQ-IN-04 | 保存操作 | ボタンクリックまたは `Ctrl+S` で保存操作を実行できること |
| 5 | REQ-IN-05 | D&amp;D 入力禁止 | アプリウィンドウ（`QWebEngineView` 領域）へのドラッグ＆ドロップ入力は受け付けず、ファイル読み込み処理を開始しないこと |

### 5.2 処理要件

#### 5.2.1 ファイルツリー取得

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-PROC-01 | ツリー取得 | 指定フォルダ配下の `.md` ファイルおよびフォルダを再帰的に取得し、ツリー構造で返すこと |
| 2 | REQ-PROC-02 | パスバリデーション | ファイルパスが指定フォルダの外を指す場合は操作を拒否し、エラーを返すこと（ディレクトリトラバーサル防止） |

#### 5.2.1a ドラッグ＆ドロップ禁止処理

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-PROC-02a | D&amp;D イベント拒否 | `QWebEngineView` アプリ領域へのドロップイベントを処理対象外とし、ファイルパス取得・読み込み処理を実行しないこと |
| 2 | REQ-PROC-02b | D&amp;D 無影響保証 | ドロップ操作実行時も現在のルートフォルダ・選択中ファイル・エディタ内容を変更しないこと |
| 3 | REQ-PROC-02c | D&amp;D 通知 | ドロップ操作が禁止である旨を UI 上に通知し、処理を中断すること |

#### 5.2.2 ファイル読み込み

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-PROC-03 | 読み込み | `.md` ファイルを読み込み、内容をエディタに表示すること |
| 2 | REQ-PROC-04 | 文字コード自動判定 | ファイルの文字コードを自動判定して読み込むこと。**Windows 環境・Shift-JIS・UTF-8 は必ず対応すること**。判定には `chardet` 等を使用し、Shift-JIS は `cp932` / `shift_jis` でカバーすること |
| 3 | REQ-PROC-04a | エンコード保持 | 読み込み時に判定したエンコードを内部に保持し、後続の保存処理で利用すること |

#### 5.2.3 ファイル保存

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-PROC-05 | 上書き保存 | 編集内容を**読み込み時に判定したエンコードで**ファイルに上書き保存すること。エンコードが不明の場合は BOM なし UTF-8 でフォールバック保存すること |
| 2 | REQ-PROC-06 | エンコード保持保存 | 保存時は元ファイルのエンコードを維持すること（例: Shift-JIS で読み込んだファイルは Shift-JIS で保存）。新規作成ファイルのデフォルトエンコードは BOM なし UTF-8 とすること |

#### 5.2.4 ファイル管理操作

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-PROC-07 | 新規ファイル作成 | 指定フォルダ内に新規 `.md` ファイルを作成できること |
| 2 | REQ-PROC-08 | ファイル削除 | 指定した `.md` ファイルを削除できること |
| 3 | REQ-PROC-09 | 名前変更 | ファイルおよびフォルダの名前を変更できること |
| 4 | REQ-PROC-10 | フォルダ外アクセス禁止 | すべてのファイル操作において、指定フォルダ外へのアクセスを禁止すること |

#### 5.2.5 Python ⇔ JavaScript 間連携

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-PROC-11 | QWebChannel 連携 | `QWebChannel` を介して JavaScript（フロントエンド）と Python（バックエンド）が直接通信すること |
| 2 | REQ-PROC-12 | HTTP サーバー不使用 | Flask 等の HTTP サーバーは使用せず、`QWebEngineView` 経由でのみ通信すること |

### 5.3 出力要件

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-OUT-01 | ファイルツリー表示 | 指定フォルダ配下の `.md` ファイル一覧をツリー形式で左ペインに表示すること |
| 2 | REQ-OUT-02 | エディタ表示 | 選択した `.md` ファイルの内容を CodeMirror エディタに表示すること |
| 3 | REQ-OUT-03 | プレビュー表示 | `marked.js` を用いて Markdown をリアルタイムで HTML レンダリングして表示すること |
| 4 | REQ-OUT-04 | 保存完了通知 | 保存完了時にユーザーへ通知（UI 上のメッセージ表示等）すること |
| 5 | REQ-OUT-05 | エラー通知 | エラー発生時（読み込み失敗・保存失敗・バリデーションエラー等）にユーザーへ通知すること |

### 5.4 画面要件（フロントエンド）

#### 5.4.1 レイアウト

```
+---------------------------+-------------------------------------------+
|        左ペイン            |              右ペイン                      |
|  ファイルツリー             | [エディタタブ] [プレビュータブ]              |
|  （フォルダ・ファイル一覧）  | または 左右分割（エディタ | プレビュー）      |
|                           |                                           |
+---------------------------+-------------------------------------------+
```

| # | 要件ID | 要件名 | 内容 |
|---|--------|--------|------|
| 1 | REQ-UI-01 | 分割レイアウト | 左ペイン（ファイルツリー）と右ペイン（エディタ＋プレビューの左右分割）の 2 カラム構成とすること |
| 2 | REQ-UI-02 | エディタ | CodeMirror を使用し、Markdown シンタックスハイライトを有効にすること |
| 3 | REQ-UI-03 | プレビュー | `marked.js` を使用し、Markdown をリアルタイムでレンダリングして表示すること |
| 4 | REQ-UI-04 | 表示切替 | 編集とプレビューを左右分割で表示すること。ユーザーがエディタ・プレビューの表示/非表示を切り替え可能なトグル機能を提供すること |
| 5 | REQ-UI-05 | 保存ボタン | 保存ボタンをUIに配置し、クリックで保存を実行できること |
| 6 | REQ-UI-06 | Ctrl+S 保存 | キーボードショートカット `Ctrl+S` で保存を実行できること |
| 7 | REQ-UI-07 | フォルダ選択 | フォルダ選択ダイアログを使用してフォルダを選択できること（選択中にパスを表示） |
| 8 | REQ-UI-08 | HTML リソース同梱 | HTML / CSS / JavaScript ファイルはアプリ内リソースとして同梱し、`QWebEngineView` にロードすること |
| 9 | REQ-UI-09 | D&amp;D 無効化 | アプリウィンドウ全体（少なくとも右ペインのエディタ・プレビュー領域）で D&amp;D によるファイルオープンを無効化すること |
| 10 | REQ-UI-10 | D&amp;D 禁止表示 | D&amp;D 操作時は「ドラッグ＆ドロップ禁止」のメッセージを UI 上に表示すること |

### 8.3 応答JSON受入基準

Python バックエンドから JavaScript フロントエンドへ返す応答は以下の形式とする。

```json
{
  "success": true,
  "data": "<操作結果データ>",
  "error": null
}
```

| フィールド | 型 | 説明 |
|------------|----|------|
| `success` | Boolean | 操作成功時 `true`、失敗時 `false` |
| `data` | Any / null | 正常時の返却データ（ツリー情報・ファイル内容等） |
| `error` | String / null | 失敗時のエラーメッセージ、正常時は `null` |

## 10. 後工程（基本設計）への引継ぎ情報

### 10.1 画面要件

| 項目 | 内容 |
|------|------|
| レイアウト | 左ペイン（ファイルツリー）＋ 右ペイン（エディタ＋プレビューの左右分割）の 2 カラム構成 |

| 項目 | 内容 |
|------|------|
| フレームワーク | PyQt6 / `QMainWindow` |
| WebView | `QWebEngineView` |

| ファイル I/O | `os` / `pathlib` で処理 |
| HTTP サーバー | 不使用 |
| 文字コード判定 | `chardet` 等を使用。Shift-JIS（`cp932`）・UTF-8 を必ずカバー |
| エンコード保持 | 読み込み時に判定したエンコードを保持し、保存時に使用。新規ファイルは UTF-8 デフォルト |
| D&amp;D ハンドリング | `QWebEngineView` またはメインウィンドウ側でドロップイベントを明示的に拒否し、読み込み処理・ツリー更新処理を呼び出さないこと |

### 10.3 フロントエンド要件

| 項目 | 内容 |

| D&amp;D JS ハンドラ | `dragover` / `drop` イベントを抑止し、Python バックエンドへのファイルパス送信を行わないこと |
