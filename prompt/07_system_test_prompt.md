# 07 システム評価プロンプト

## 目的
要件がすべて実装されているかを最終評価し、リリース可否を判定する。

## 入力
- `project/document/01_****.md`
- 実装一式（`project/src/`）
- 結合評価結果（`project/src/test/`）

## 指示
1. 要件トレーサビリティに沿って総合評価する。
2. 正常系・異常系の最終判定を実施する。
3. 動作環境条件（Windows 10/11、PyInstaller生成 .exe 単一ファイル、インストール不要）での成立性を確認する。
4. READMEの利用手順と実装の整合性を確認する。
5. 完全ローカル動作（外部サーバ接続不要）とスタンドアロン性を最終確認する。
6. リリース可否と残課題を明示する。

## 出力形式
- システム評価記録（Markdown）を `project/src/test/` に保存
- 最終判定サマリ（OK/条件付きOK/NG）を記載

## 停止条件
- 重大不具合または要件未達がある場合は停止し、差戻し工程を明示する。

## 工程ゲート（完了承認確認）
- 評価成果物生成後、完了判定の前にGitHub使用者へチャットベースのレビューを実施する。
- チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
- GitHub使用者が承認した場合にプロジェクト完了とする。未承認の場合は07工程で修正し再確認を実施する。
