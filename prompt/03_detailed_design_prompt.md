# 03 詳細設計プロンプト

## 目的
基本設計をもとに、実装可能な詳細仕様をフロントエンド/バックエンド別に作成する。

## 入力
- `project/document/02_****.md`
- `project/document/01_****.md`

## 指示
1. Python側の各関数仕様を関数レベルで定義する。
   - ツリー取得・ファイル読み込み（`chardet` 文字コード自動判定）・UTF-8保存・新規作成・削除・名前変更
   - フォルダ外アクセス禁止バリデーション
   - `QWebChannel` 経由で公開するメソッド・JavaScriptへのシグナル仕様
2. JavaScript側（HTML / CSS / JS）の各モジュール仕様を定義する。
   - CodeMirror 初期化・エディタ操作（Markdownシンタックスハイライト）
   - marked.js プレビューレンダリング
   - ファイルツリー表示・選択・操作（新規・削除・名前変更）
   - Ctrl+S ショートカット・保存ボタンのイベントハンドキング
3. 異常系処理仕様（パス不正、ファイル未存在、文字コード読み取り失敗、保存失敗）を定義する。
4. PyInstaller ビルド設定（同梱リソース、.spec 要件）を定義する。
5. 単体評価観点にトレース可能な設計IDを付与する。

## 出力形式
- Markdown
- 見出し構成: Python側設計 / JavaScript側設計 / QWebChannelインターフェース仕様 / 異常系仕様 / PyInstaller設定 / テスト観点
- 保存先: `project/document/03_****.md`

## 停止条件
- 02成果物で実装判断できない箇所がある場合は停止し、改訂要求を列挙する。

## 工程ゲート（次工程進行確認）
- 成果物生成後、04工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
- チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
- GitHub使用者が承認した場合のみ04工程へ進む。未承認の場合は03工程で修正し再確認を実施する。
