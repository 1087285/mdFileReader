# 02 基本設計プロンプト

## 目的
要件定義成果物から、アプリ全体の構成・機能配置・データフローを定義する。

## 入力
- `project/document/01_****.md`
- `requests/minutes_v*.md` のうち版数が最大のファイル（最新議事録）

## 指示
1. システム構成（PyQt6 QMainWindow / QWebEngineView + QWebChannel / JavaScript）を定義する。
2. アプリケーションフロー（フォルダ選択→ツリー表示→ファイル選択→表示/編集→保存）を図示する。
3. Python側とJavaScript側の責務を分離する。
   - Python: ファイルI/O（`os`/`pathlib`）、文字コード自動判定、フォルダ外アクセス禁止バリデーション
   - JavaScript: CodeMirror エディタ、marked.js プレビュー、Ctrl+S 保存ショートカット
4. QWebChannel を介したPython ⇔ JavaScript 連携インターフェース（メソッド・シグナル）を定義する。
5. PyInstaller ビルド・配布フロー（.exe 単一ファイル化、Windows 10/11）を定義する。
6. 詳細設計へ渡すコンポーネント境界・インターフェース情報を整理する。

## 出力形式
- Markdown
- 図表中心（システム構成図、アプリケーションフロー図、責務分離表、連携インターフェース一覧）
- 保存先: `project/document/02_****.md`

## 停止条件
- 01成果物に矛盾・不足がある場合は停止し、差戻し項目を列挙する。

## 工程ゲート（次工程進行確認）
- 成果物生成後、03工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
- チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
- GitHub使用者が承認した場合のみ03工程へ進む。未承認の場合は02工程で修正し再確認を実施する。
