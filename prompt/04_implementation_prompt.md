# 04 実装プロンプト

## 目的
詳細設計に従って、Python実装（PyQt6）とJavaScript/HTML/CSS実装を行う。

## 入力
- `project/document/03_****.md`
- 参照: `project/document/02_****.md`, `project/document/01_****.md`

## 指示
1. `project/src/backend/` にPython実装を配置する。
   - `QMainWindow` / `QWebEngineView` / `QWebChannel` を使ったアプリケーション本体
   - ツリー取得・読み込み（`chardet`）・UTF-8保存・新規作成・削除・名前変更各関数
   - フォルダ外アクセス禁止バリデーション
   - `QWebChannel` 経由でJSへ公開するメソッド・シグナル
2. `project/src/frontend/` にJavaScript/HTML/CSS実装を配置する。
   - CodeMirror エディタ（Markdownシンタックスハイライト）
   - marked.js プレビュー
   - 左右ペインレイアウト（ファイルツリー / エディタ＋プレビュー）
   - Ctrl+S ショートカット・保存ボタンのイベントハンドラ
3. 実装は設計トレース可能な単位で分割する。
4. 設計との差異がある場合は差異理由を記録する。
5. 次工程（単体評価）に必要な入出力仕様を明記する。
6. 03詳細設計の変更点は `project/src/` の該当コードへ反映し、反映ファイルを列挙する。
7. PyInstaller による .exe 単一ファイル化のビルド手順を実装記録へ明記する。
8. 実装後は最低限の動作確認（対象機能の実行または単体テスト）を行い、結果を記録する。

## 出力形式
- 実装コード一式
- 実装メモ（Markdown）：`project/document/04_****.md`

## 停止条件
- 詳細設計に不足がある場合は実装を停止し、03へ差戻し事項を列挙する。

## 工程ゲート（次工程進行確認）
- 実装成果物生成後、05工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
- チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
- GitHub使用者が承認した場合のみ05工程へ進む。未承認の場合は04工程で修正し再確認を実施する。
