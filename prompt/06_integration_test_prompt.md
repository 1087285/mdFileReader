# 06 結合評価プロンプト

## 目的
基本設計通りに、サブシステム間の連携が成立しているかを評価する。

## 入力
- `project/document/02_****.md`
- `project/src/`
- 単体評価結果（`project/src/test/`）

## 指示
1. アプリケーションE2Eフロー（フォルダ選択→ツリー表示→ファイル選択→表示/編集→保存）を評価する。
2. Python⇔JavaScript間の `QWebChannel` 連携（メソッド呼び出し・シグナル送受信）が設計どおり動作するかを評価する。
3. 境界条件（空フォルダ・大量ファイル・非UTF-8ファイル・長パス・特殊文字ファイル名）を評価する。
4. 異常系連携（パス不正・保存失敗時のUI通知・フォルダ外アクセス時のブロック）を評価する。
5. 不具合の再現手順と影響範囲を記録する。
6. システム評価へ引き継ぐ課題を整理する。

## 出力形式
- 結合評価記録を `project/src/test/` に保存

## 停止条件
- 基本設計とのトレース不能な仕様差異がある場合は停止し、差戻し先を明記する。

## 工程ゲート（次工程進行確認）
- 評価成果物生成後、07工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
- チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
- GitHub使用者が承認した場合のみ07工程へ進む。未承認の場合は06工程で修正し再確認を実施する。
