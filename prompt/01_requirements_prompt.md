# 01 要件定義プロンプト

## 目的
`requests/minutes_v*.md` のうち版数が最大のファイル（最新議事録）をもとに、要件定義成果物（`project/document/01_****.md`）を作成する。

## 入力
- `requests/minutes_v*.md` のうち版数が最大のファイル（最新議事録）

## 指示
1. 要求・制約・前提を分類して整理する。
2. 機能要件/非機能要件/運用要件/保守要件を定義する。
3. 入出力仕様を定義する。
   - 入力: ユーザが指定するフォルダパス（MDファイル格納先）
   - 出力: MDファイルの表示・編集・保存結果（UTF-8書き込み）
4. ファイル操作機能の要件を明記する（ツリー取得・読み込み・保存・新規作成・削除・名前変更・バリデーション）。
5. UI要件を定義する（左右ペインレイアウト、CodeMirror エディタ、marked.js プレビュー、Ctrl+S 保存）。
6. 配布形式（PyInstaller による .exe 単一ファイル、Windows 10/11、インストール不要）を明記する。
7. 正常系/異常系の受入観点を列挙する。
8. 後工程（基本設計）が必要とする情報を明示する。

## 出力形式
- Markdown
- 見出し構成: 目的 / スコープ / 要件一覧 / 入出力仕様 / UI要件 / 制約・前提 / 配布形式 / 受入条件 / 未決事項
- 保存先: `project/document/01_****.md`

## 停止条件
- 要件判定に必要な情報が不足している場合は処理を停止し、追加確認事項を列挙する。

## 工程ゲート（次工程進行確認）
- 成果物生成後、02工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
- チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
- GitHub使用者が承認した場合のみ02工程へ進む。未承認の場合は01工程で修正し再確認を実施する。
