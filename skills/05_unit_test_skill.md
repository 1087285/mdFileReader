# 05 単体評価スキル

## 目的
実装が詳細設計を満たすかをモジュール単位で評価し、関数ごとにMC/DC（Modified Condition/Decision Coverage）100%達成を確認する。

## 入力
- 詳細設計
- 実装コード

## 手順
1. Python側・JavaScript側それぞれのモジュールからテストケース（正常/異常）を作成する。
   - Python: ツリー取得・読み込み・保存・新規作成・削除・名前変更・バリデーション・`QWebChannel`メソッド
   - JavaScript: CodeMirror エディタ操作・marked.js プレビュー・Ctrl+S／保存ボタンイベント
   - 異常系: パス不正・ファイル未存在・文字コード読み取り失敗・保存失敗
2. 条件分岐を持つ各関数で、基本条件ごとのMC/DCケース対を作成し、関数単位で達成率100%を確認する。
3. 期待結果を明文化して実行する。
4. 実績との差分を記録し、原因工程を推定する。
5. 結合評価へ未解決事項（MC/DC未達関数を含む）を引き継ぐ。
6. 成果物生成後、次工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
7. チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
8. GitHub使用者の承認後に次工程へ進行する。

## 成果物
- 単体評価記録
- 関数別MC/DC達成結果（達成/未達、未達理由）
- 不具合一覧

## 品質チェック
- 正常系と異常系の双方を網羅している。
- 関数ごとのMC/DC達成率が100%である（未達時は理由と対策が記録されている）。
- ケースと仕様IDの対応が追跡可能である。
