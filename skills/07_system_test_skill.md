# 07 システム評価スキル

## 目的
全要件に対する最終達成度を評価し、リリース可否を判定する。

## 入力
- 要件定義
- 実装コード
- 結合評価結果

## 手順
1. 要件トレーサビリティを作成する。
2. 正常系/異常系で最終判定を実施する。
3. 動作環境条件（Windows 10/11、PyInstaller生成 .exe 単一ファイル、インストール不要）での成立性を確認する。
4. 完全ローカル動作（外部サーバ接続不要）とスタンドアロン性を最終確認する。
5. 残課題とリスクを明示して可否判定する。
6. 成果物生成後、完了判定の前にGitHub使用者へチャットベースのレビューを実施する。
7. チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
8. GitHub使用者の承認後に完了とする。

## 成果物
- システム評価報告
- リリース判定

## 品質チェック
- 要件未評価項目がない。
- 判定根拠が証跡に紐づいている。
