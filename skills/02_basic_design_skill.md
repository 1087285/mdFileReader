# 02 基本設計スキル

## 目的
要件をシステム構成・責務分割・データフローへ変換する。

## 入力
- 要件定義成果物

## 手順
1. システム構成（PyQt6 QMainWindow / QWebEngineView + QWebChannel / JavaScript）を定義する。
2. アプリケーションフロー（フォルダ選択→ツリー表示→ファイル選択→表示/編集→保存）を図示する。
3. Python側とJavaScript側の責務境界を分離する。
   - Python: `os`/`pathlib` によるファイルI/O、文字コード自動判定、フォルダ外アクセス禁止バリデーション
   - JavaScript: CodeMirror エディタ（Markdownシンタックスハイライト）、marked.js プレビュー、Ctrl+S 保存
4. QWebChannel を介した連携インターフェース（呼び出しメソッド・Pythonシグナル）を定義する。
5. PyInstaller ビルドフロー（.exe 単一ファイル化）と配布方法を設計へ組み込む。
6. 成果物生成後、次工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
7. チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
8. GitHub使用者の承認後に次工程へ進行する。

## 成果物
- システム構成図
- アプリケーションフロー図
- コンポーネント責務分離表
- Python ⇔ JavaScript 連携インターフェース一覧

## 品質チェック
- 全要件に対する設計要素が対応している。
- 詳細設計へ受け渡し可能な境界が定義されている。
