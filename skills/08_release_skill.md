# 08 リリーススキル

## 目的
07システム評価の確定結果をもとに、リリース向け文書を最終化し、出荷可否を判定する。

## 入力
- システム評価記録
- 最終サマリ
- リリースノート
- プッシュ運用ガイド
- `README.md`（利用手順・動作環境）
- `skills/08_release_skill.md` 自身（手順・品質チェックの準拠元）

## 手順
1. 07工程の判定結果・条件付き項目・未解決課題を整理する。
2. リリースノート（A2）を最新化する。
3. プッシュ運用ガイド（A3）を最新化する。
4. プロジェクト最終サマリ（A1）へ工程01〜07結果を統合する。
5. `README.md` と .exe 配布物の整合（起動手順・Windows 10/11・インストール不要）を確認する。
6. 条件付き項目（未確認環境・残存不具合）の残タスクを明記する。
7. 出荷判定（Go / Conditional Go / No-Go）を決定する。
8. 成果物生成後、完了判定の前にGitHub使用者へチャットベースのレビューを実施する。
9. チャットでは変更前の状態、変更後の状態と変更の考え方、懸念事項・特記事項（ある場合）、判断を仰ぎたい質問（ある場合）を説明する。
10. GitHub使用者の承認後、pushまで進めるかを確認し、明示的な合意を得る。
11. 合意後、リリースタグ（例: `vX.Y.Z`）を作成して push する。
12. GitHub Releases 本体を作成し、`A2_release_notes.md` をリリースノート本文として反映する。PyInstaller生成 .exe を配布物としてアッチメントする。
13. 生成されたリリースURLを `A1_project_final_summary.md` と `A2_release_notes.md` に追記して整合を確認する。
14. GitHub使用者の承認後に完了とする。

## 成果物
- A1 最終サマリ
- A2 リリースノート
- A3 プッシュ運用ガイド
- リリースタグ（`vX.Y.Z`）
- GitHub Releases 本体（PyInstaller生成 .exe アッチメント付き）
- リリースURL反映済みの A1/A2
- 出荷判定（Go / Conditional Go / No-Go）

## 品質チェック
- A1/A2/A3、README.md、.exe 配布物に矛盾がない。
- 出荷判定の根拠が証跡に紐づいている。
- 条件付き項目と残タスクが明示されている。
- リリースタグと GitHub Releases 本体が作成済みで、PyInstaller生成 .exe がアッチメントされ、公開URLが A1/A2 に反映されている。
