# HOWTOUSE

## 1. 目的
本書は、`topDownTest` アプリの利用手順を利用者向けに定義する。

## 2. 事前準備
- フロントエンド（`project/src/frontend`）をWeb公開ディレクトリに配置する
- バックエンド（`project/src/backend`）を実行可能な場所へ配置する
- Python実行環境を準備する
- 必要に応じてメール通知環境変数を設定する（`SMTP_HOST`, `SMTP_PORT`, `SMTP_FROM`）

## 3. 入力
- フォルダパス（必須）
- メールアドレス（必須）

入力値は UTF-8 と Shift-JIS（Windows-31J/CP932）を受け付ける。
Windows 環境では、フォルダパスに空白を含む指定も受け付ける。

## 4. 実行手順
1. `index.php` にアクセスする
2. フォルダパスとメールアドレスを入力し、決定ボタンを押す
3. 確認ダイアログで入力値を確認する
4. OK を押して処理を開始する（キャンセル時は入力保持で元画面へ戻る）

## 5. 実行結果
- `result.csv` が生成される
- 一致結果の集約コピー先フォルダが生成される
- 配置先はいずれも「ユーザ指定パスと同一階層」
- 同名フォルダが既存の場合は、`copied_results_1`, `copied_results_2` ... を採番して新規作成し、既存フォルダは上書きしない

## 6. 異常時
- 入力不正（パス/メール）時は処理を停止し、エラーを表示する
- 文字コード判定不能時は `INVALID_ENCODING` を返す
- 応答は標準JSON（`status`, `error_code`, `message`, `run_id`）に準拠する

## 7. 運用
- 要求・要件・出力仕様が変更された場合は、本書の更新要否を確認し、必要に応じて更新する
- デプロイ詳細は `project/document/Z0_deploy_checklist_windows_xampp.md` を参照する
- 08工程の最終更新時は、GitHub使用者の承認後に「プッシュまで進めるか」を確認し、明示的な合意後に `git push` を実施する
