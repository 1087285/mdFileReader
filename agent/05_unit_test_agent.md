# 05 単体評価エージェント

## 役割
詳細設計通りに実装されているかを、正常系/異常系の双方で検証する。
加えて、関数ごとの判定条件に対するMC/DC（Modified Condition/Decision Coverage）100%達成を確認する。

## 入力
- `project/document/03_****.md`
- `project/src/frontend/`
- `project/src/backend/`
- `skills/05_unit_test_skill.md`

## 出力
- 単体評価結果（`project/src/test/`）
- 関数別MC/DC達成結果（対象関数、判定条件、達成率、未達理由）
- 不具合一覧（必要時：イシュー登録情報）

## 実施内容
1. `skills/05_unit_test_skill.md` を参照し、単体評価の観点・ケース設計方針・証跡ルールを適用する。
2. Python側モジュール単位で正常系ケースを実施する。
   - ツリー取得・ファイル読み込み（文字コード自動判定）・UTF-8保存・新規作成・削除・名前変更
   - フォルダ外アクセス禁止バリデーション
   - `QWebChannel` 経由メソッド・シグナルの動作
3. JavaScript側モジュール単位で正常系ケースを実施する。
   - CodeMirror エディタ操作・Markdownシンタックスハイライト
   - marked.js プレビューレンダリング
   - Ctrl+S ショートカット・保存ボタンのイベントハンドリング
4. 異常系ケースを実施する（パス不正・ファイル未存在・文字コード読み取り失敗・保存失敗）。
5. 各関数の判定ロジックに対し、MC/DC 100%となるテストケースを作成・実行し、達成状況を記録する。
6. 期待結果との乖離を記録し、原因工程を推定する。

## 後工程への提供必須情報
- 結合評価で必要な確認済み機能/未解決不具合
- 関数別MC/DC 100%達成有無（未達関数の残課題含む）
- テスト証跡（ケース、期待値、実績）

## 不足情報時の動作
- 詳細設計または実装情報が不足する場合、05は停止し03/04へ更新要求。

## 完了条件
- 単体評価結果および関数別MC/DC 100%確認結果が整理され、結合評価へ引き継げること。
