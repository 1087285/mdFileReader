# 02 基本設計エージェント

## 役割
要件定義成果物をもとに、アプリ全体の機能配置とコンポーネント境界を定義する。

## 入力
- `project/document/01_****.md`
- `requests/minutes_v*.md` のうち版数が最大のファイル（最新議事録）
- `skills/02_basic_design_skill.md`

## 出力
- 基本設計成果物（`project/document/02_****.md`）
- 詳細設計へ渡す構成定義

## 実施内容
1. `skills/02_basic_design_skill.md` を参照し、基本設計の観点・記法・成果物粒度を適用する。
2. システム構成（PyQt6 QMainWindow / QWebEngineView + QWebChannel / JavaScript）を定義する。
3. アプリケーションフロー（フォルダ選択→ツリー表示→ファイル選択→表示/編集→保存）を定義する。
4. Python側（ファイルI/O・バリデーション）とJavaScript側（UI描画・エディタ・プレビュー）の責務境界を明確化する。
5. QWebChannel を介したPython ⇔ JavaScript間のデータ連携インターフェースを定義する。
6. PyInstaller によるビルド・配布フロー（.exe 単一ファイル化）の配置を明確化する。

## 後工程への提供必須情報
- 画面レイアウトとUI責務（左ペイン：ファイルツリー / 右ペイン：エディタ＋プレビュー）
- Python ⇔ JavaScript 連携インターフェース（QWebChannel 経由のメソッド・シグナル定義）
- ファイル操作責務（Python側：`os`/`pathlib` で処理、文字コード自動判定、フォルダ外アクセス禁止）
- UI責務（JavaScript側：CodeMirror エディタ、marked.js プレビュー、Ctrl+S 保存）
- エラー時停止・イシュー登録フロー

## 不足情報時の動作
- 01成果物に矛盾・不足がある場合、02は処理を停止し01へ改訂要求を返す。

## 完了条件
- 詳細設計で実装単位に分解できる粒度になっていること。
