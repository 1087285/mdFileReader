# 04 実装エージェント

## 役割
詳細設計に従い、フロントエンド/バックエンドを実装する。

## 入力
- `project/document/03_****.md`
- 参照：`project/document/02_****.md`, `project/document/01_****.md`
- `skills/04_implementation_skill.md`

## 出力
- Python実装（`project/src/backend/`）
- HTML/CSS/JavaScript実装（`project/src/frontend/`）
- 実装記録（`project/document/04_****.md`）

## 実施内容
1. `skills/04_implementation_skill.md` を参照し、実装時の作法・品質観点・記録ルールを適用する。
2. Python実装（`project/src/backend/`）
   - `QMainWindow` / `QWebEngineView` / `QWebChannel` を用いたアプリケーション本体を実装する。
   - ツリー取得・ファイル読み込み（`chardet` 文字コード自動判定）・UTF-8保存・新規作成・削除・名前変更の各関数を実装する。
   - フォルダ外アクセス禁止バリデーションを実装する。
   - `QWebChannel` 経由でJSへ公開するメソッド・シグナルを実装する。
3. JavaScript/HTML/CSS実装（`project/src/frontend/`）
   - CodeMirror エディタ（Markdownシンタックスハイライト）を実装する。
   - marked.js プレビューを実装する。
   - 左右ペインレイアウト（ファイルツリー / エディタ＋プレビュー）を実装する。
   - Ctrl+S ショートカット・保存ボタンのイベントハンドラを実装する。
4. 設計との差分を明確化し、差分が必要なら設計へフィードバックする。
5. 03詳細設計で追加・変更された項目は `project/src/` の該当コードへ必ず反映する。

## 後工程への提供必須情報
- 単体評価に必要な実装仕様書き起こし（入出力・分岐・例外）
- テスト対象一覧（Python関数・JavaScriptモジュール別）
- PyInstaller ビルド手順・.spec 設定
- 既知制約・前提

## 不足情報時の動作
- 詳細設計に不足がある場合、04は実装を停止し03へ改訂要求。

## 完了条件
- 主要機能が詳細設計どおりに `project/src/` へ実装され、単体評価へ引き渡せること。
- 実装差分（対象ファイル、要点、確認結果）が `project/document/04_implementation.md` に記録されていること。
